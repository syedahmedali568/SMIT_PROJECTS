Inline Elements

  • <b> (bold): Makes text bold.
    Use case: Emphasize important keywords or phrases within a sentence.
  • <i> (italic): Makes text italicized.
    Use case: Highlight titles of books, movies, or foreign words.
  • <a> (anchor): Creates a hyperlink to another webpage or section within the current page.
    Use case: Link text to relevant resources or internal navigation.
  • <strong> (strong): Indicates strong importance of text.
    Use case: Highlight crucial information or main points.
  • <em> (emphasis): Indicates emphasis on text.
    Use case: Emphasize a specific point within a sentence without needing italics.
  • <span> (generic inline container): Applies styles to a section of text without semantic meaning.
    Use case: Highlight specific text with custom background color or font.
  • <img> (image): Inserts an image into the webpage.
    Use case: Display logos, product photos, or illustrative images.
  • <input> (form input): Creates an input field for user interaction text, checkbox, etc..
    Use case: Collect user data in forms (login information, search queries).
  • <button> (button): Creates a clickable button to trigger actions.
    Use case: Submit forms, activate functionalities, or navigate between pages.
  • <code> (code): Displays text as inline code with a monospaced font.
    Use case: Show snippets of code within sentences.

Block-Level Elements

  • <p> (paragraph): Defines a paragraph of text.
    Use case: Build the main content body of a webpage.
  • <h1> - <h6> (headings): Define headings of different levels.
    Use case: Structure the document hierarchy and improve readability.
  • <div> (division): Creates a generic division for content with no specific meaning.
    Use case: Group related content together e.g., sidebars, content sections.
  • <ul> (unordered list): Creates an unordered list of items.
    Use case: List items with no specific order (e.g., ingredients, features).
  • <ol> (ordered list): Creates an ordered list of items with sequential numbering.
    Use case: List steps in a process, instructions, or ranked items.
  • <blockquote> (quotation): Defines a block of quoted text.
    Use case: Display quotations or excerpts.
  • <table> (table): Creates a tabular data structure.
    Use case: Present data in a structured format with rows and columns.
  • <figure> (figure): Groups an image or other content with its caption.
    Use case: Display images with captions or infographics.
  • <section> (section): Defines a self-contained section of content.
    Use case: Structure complex webpages with thematic subsections.
  • <form> (form): Defines a collection of input elements for user interaction.
    Use case: Create forms for login, registration, or data collection.
